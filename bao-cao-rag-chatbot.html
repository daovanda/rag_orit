Chatbot RAG hỗ trợ tra cứu tài liệu Zilcode

Dự án xây dựng một trợ lý AI chạy trên Cloudflare Worker, có khả năng chat thông thường, tra cứu tài liệu Zilcode bằng RAG, gọi tool theo ngữ cảnh và tạo ảnh biểu đồ/sơ đồ khi người dùng yêu cầu.

Mục Tiêu Bài Làm

Tạo một chatbot có thể trả lời câu hỏi về Zilcode dựa trên tài liệu nội bộ, đồng thời vẫn giữ khả năng hội thoại bình thường và mở rộng qua tool.

Tra cứu đúng tài liệu

Tài liệu Markdown được chunk, embed và lưu vào Vectorize/KV để model trả lời dựa trên nội dung đã ingest.

Không ép mọi câu hỏi vào RAG

Câu chào hỏi, câu hỏi ngoài Zilcode hoặc câu hỏi về năng lực trợ lý được xử lý bằng general_chat.

Mở rộng bằng tool

Bổ sung tool draw_chart để tạo ảnh biểu đồ/sơ đồ, và chuẩn bị sẵn mock tool cho workflow/context.

Kiến Trúc Tổng Quan

Worker đóng vai trò backend API và agent orchestrator. Vectorize tìm đoạn tài liệu liên quan, KV lưu nội dung chunk, Workers AI chạy các model.

1

Người dùng

Gửi câu hỏi từ chat.html hoặc app bên ngoài.

2

Worker API

Nhận POST /chat, gọi vòng lặp agentic.

3

Tool Selection

CHAT_MODEL quyết định dùng tool phù hợp.

4

RAG / Chat / Image

Gọi Vectorize, KV, general chat hoặc Flux tùy yêu cầu.

5

Tổng hợp

Model tạo câu trả lời cuối, làm sạch Markdown nếu là RAG.

6

Response

Trả về text, danh sách tool và ảnh nếu có.

Thành phần chính

  • src/index.ts: Cloudflare Worker, API, tool definitions, agentic loop.
  • scripts/ingest.ts: đọc tài liệu, chunk, embed, upsert vào Vectorize và KV.
  • chat.html: giao diện thử nghiệm API, hiển thị text, badge tool và ảnh.
  • doc/*.md: nguồn tài liệu tiếng Việt về Zilcode.

Dịch vụ Cloudflare sử dụng

  • Workers: chạy backend serverless.
  • Workers AI: chạy Llama, BGE-M3 và Flux.
  • Vectorize: lưu embedding để tìm kiếm ngữ nghĩa.
  • KV Namespace: lưu nội dung thật của từng chunk tài liệu.

Pipeline RAG

RAG được tách thành hai giai đoạn: ingest tài liệu trước, sau đó truy vấn khi người dùng chat.

Giai đoạn ingest

  1. Đọc các file doc/*.md.
  2. Chuẩn hóa newline, lấy tiêu đề và xác định loại tài liệu.
  3. Chunk theo heading Markdown từ # đến ######.
  4. Nếu section dài hơn 1800 ký tự, chia theo block, câu và overlap 160 ký tự.
  5. Embed bằng @cf/baai/bge-m3.
  6. Lưu vector vào Vectorize, lưu text chunk vào KV.

Giai đoạn hỏi đáp

  1. Người dùng hỏi về Zilcode.
  2. CHAT_MODEL chọn rag_search.
  3. Worker embed câu hỏi bằng BGE-M3.
  4. Vectorize tìm các chunk gần nghĩa nhất.
  5. Worker lấy nội dung chunk từ KV.
  6. Model tổng hợp câu trả lời cuối dựa trên tài liệu.
Vectorize không lưu nội dung dài của tài liệu; nó dùng để tìm đúng chunk bằng vector. KV lưu text thật của chunk để model đọc và trả lời. Hai thành phần này kết hợp thành document retrieval layer.

API Cho App Bên Ngoài

App khác có thể tích hợp qua endpoint chat của Worker. Response có thể chứa text và ảnh. 

POST https://ragorit.daovanda2405.workers.dev/chat
Content-Type: application/json

{
  "message": "Zilcode là gì?"
}
{
  "success": true,
  "response": "Zilcode là một nền tảng no-code...",
  "tools_called": ["rag_search"],
  "images": []
}

Request có context

App có thể gửi thêm context để tool get_screen_context hiểu màn hình hiện tại.

screen selected_node resource_id

Bảo mật API

Nên thêm API key qua Cloudflare Secret như CHAT_API_KEY, sau đó kiểm tra header x-api-key trong Worker để chỉ người được cấp key mới gọi được API.

Kết Quả Đã Làm Được

Các phần chính đã hoàn thành và có thể demo trực tiếp qua UI hoặc API.

  • Đã ingest tài liệu Markdown tiếng Việt.
  • Search theo nghĩa, không phụ thuộc hoàn toàn vào keyword.
  • Trả lời dựa trên chunk tài liệu liên quan.

Tool-calling ổn định hơn

  • Không ép mọi câu hỏi phải gọi RAG.
  • Có guard tránh general_chat ghi đè kết quả RAG.
  • Giới hạn token để tránh câu trả lời bị cắt.

UI test được

  • Gửi chat tới Worker URL.
  • Hiển thị tool đã gọi.
  • Hiển thị ảnh sinh bởi tool draw_chart.

Hướng Phát Triển

Những việc nên làm tiếp nếu dự án đi từ bài tập sang thử nghiệm nội bộ hoặc production.

Bảo mật

Thêm API key, giới hạn CORS theo domain, rate limit và log request để tránh bị gọi ngoài ý muốn.

Chất lượng RAG

Thêm reranking, lưu nguồn trích dẫn, đánh giá bộ câu hỏi chuẩn và tối ưu chunk theo từng loại tài liệu.

Tích hợp Zilcode thật

Thay mock get_workflow và get_screen_context bằng API/context thật từ ứng dụng.
